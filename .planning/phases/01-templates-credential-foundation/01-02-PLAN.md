### {{.TASKFILE_DIR}} resolution
`{{.TASKFILE_DIR}}` resolves to the directory containing the INCLUDED Taskfile, not the root.
When a per-service Taskfile is installed to `.devtools/redis.Taskfile.yml` and the root
`.devtools/Taskfile.yml` includes it, `{{.TASKFILE_DIR}}` in `redis.Taskfile.yml` resolves
to `.devtools/`. This is the correct, tested behavior.

### Taskfile version
All Taskfiles use `version: '3'` (schema string, not number). This is the current and only
supported schema. The `includes` feature with `optional: true` requires v3.17+.

### Profile activation strategy
- `up` task: does NOT pass `--profile services` explicitly — relies on `COMPOSE_PROFILES=services` in .env
- `up-ui` task: passes `--profile ui` (Docker Compose merges with COMPOSE_PROFILES from .env)
- `up-monitoring` task: passes `--profile monitoring` (merged with COMPOSE_PROFILES from .env)
- `down` task: no `--profile` needed (stops all containers across all profiles)

### Compose file naming
Compose files installed to `.devtools/` will be named `<service>.compose.yml`.
The `-f {{.TASKFILE_DIR}}/<service>.compose.yml` path is correct for installed files.
Template source is `compose-templates/<service>/<service>.compose.yml`.
Installed target is `.devtools/<service>.compose.yml` (SKILL handles the copy).

### Service name in compose
The docker compose service name inside each compose file is:
- redis, redisinsight, redis_exporter
- rabbitmq, rabbitmq_exporter
- postgres, pgadmin, postgres_exporter
- mysql, phpmyadmin, mysql_exporter
- mongodb, mongo_express, mongodb_exporter
- prometheus, grafana
Use these exact names in `logs` and `restart` task targets.

### Root Taskfile global tasks
The root Taskfile provides global orchestration tasks (up, down, up-ui, up-monitoring)
that call all service tasks with `ignore_error: true` so missing services are silently skipped.
This requires calling each service's namespace task individually (no `for:` loop — compatibility).

### FILE: taskfile-templates/redis/redis.Taskfile.yml

```yaml
# Redis Taskfile template
# Installed to: .devtools/redis.Taskfile.yml
# Included by: .devtools/Taskfile.yml as namespace 'redis'
# Usage: task redis:up | task redis:down | task redis:logs | task redis:restart
# {{.TASKFILE_DIR}} resolves to .devtools/ when installed — same dir as redis.compose.yml

version: '3'

tasks:

  up:
    desc: Start Redis (main service — requires COMPOSE_PROFILES=services in .devtools/.env)
    cmds:
      - docker compose -f {{.TASKFILE_DIR}}/redis.compose.yml up -d

  up-ui:
    desc: Start Redis + RedisInsight UI (adds ui profile)
    cmds:
      - docker compose -f {{.TASKFILE_DIR}}/redis.compose.yml --profile ui up -d

  up-monitoring:
    desc: Start Redis + metrics exporter (adds monitoring profile)
    cmds:
      - docker compose -f {{.TASKFILE_DIR}}/redis.compose.yml --profile monitoring up -d

  down:
    desc: Stop Redis and all its companions
    cmds:
      - docker compose -f {{.TASKFILE_DIR}}/redis.compose.yml down

  logs:
    desc: Follow Redis container logs
    cmds:
      - docker compose -f {{.TASKFILE_DIR}}/redis.compose.yml logs -f redis

  restart:
    desc: Restart Redis container
    cmds:
      - docker compose -f {{.TASKFILE_DIR}}/redis.compose.yml restart redis
```

Create `taskfile-templates/root/Taskfile.yml`. This is the template that the SKILL installs
as `.devtools/Taskfile.yml` in the user's project.

Key design decisions:
1. `includes:` lists all 6 services with `optional: true` — per-service Taskfiles are optional
   because not all services will be installed. Missing included files are silently ignored.
2. Global tasks (up, down, up-ui, up-monitoring) call each service's task with `ignore_error: true`
   so missing namespaces (service not installed) are silently skipped.
3. The `dir:` property on each include is NOT set here — the root Taskfile is installed TO
   `.devtools/Taskfile.yml` and all service Taskfiles are in the same directory, so relative
   paths in includes (`./redis.Taskfile.yml`) work correctly when installed.
4. No `dir:` override needed — {{.TASKFILE_DIR}} in included files resolves to `.devtools/`
   when the root Taskfile is at `.devtools/Taskfile.yml`.

# Root Taskfile template
# Installed to: .devtools/Taskfile.yml
# Add to your project root Taskfile.yml:
#   includes:
#     devtools:
#       taskfile: ./.devtools/Taskfile.yml
#       optional: true
# Then access services as: task devtools:redis:up

version: '3'

includes:
  redis:
    taskfile: ./redis.Taskfile.yml
    optional: true    # Silently ignored if Redis is not installed
  rabbitmq:
    taskfile: ./rabbitmq.Taskfile.yml
    optional: true
  postgres:
    taskfile: ./postgres.Taskfile.yml
    optional: true
  mysql:
    taskfile: ./mysql.Taskfile.yml
    optional: true
  mongodb:
    taskfile: ./mongodb.Taskfile.yml
    optional: true
  monitoring:
    taskfile: ./monitoring.Taskfile.yml
    optional: true

tasks:

  up:
    desc: Start all installed services (skips services not yet installed)
    cmds:
      - task: redis:up
        ignore_error: true
      - task: rabbitmq:up
        ignore_error: true
      - task: postgres:up
        ignore_error: true
      - task: mysql:up
        ignore_error: true
      - task: mongodb:up
        ignore_error: true

    desc: Stop all installed services

      - task: redis:down
        ignore_error: true

        ignore_error: true
      - task: postgres:down
        ignore_error: true
      - task: mysql:down
        ignore_error: true

        ignore_error: true

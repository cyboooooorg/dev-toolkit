Create `compose-templates/.env.example`. This file serves two purposes:
1. Documents all env vars for the user (readable reference)
2. Is copied by the SKILL to `.devtools/.env`, with `{{TOKEN}}` values substituted by the skill

IMPORTANT CONVENTION: Values in .env.example use `{{TOKEN}}` syntax for skill-substituted
values (e.g., `REDIS_PORT={{PORT}}`). The SKILL replaces `{{PORT}}` with the user's chosen
value. Fixed/non-substituted values (like COMPOSE_PROFILES) are written literally.

The file MUST include:
- A preamble comment explaining the format
- `COMPOSE_PROJECT_NAME={{PROJECT_NAME}}` — SKILL substitutes with user's project name
- `COMPOSE_PROFILES=services` — activates the services profile so containers start by default
- One section per service with all vars that appear in that service's compose file
- `{{TOKEN}}` placeholders for all values the skill prompts the user to configure
- Default values in comments for reference
- A `.gitignore reminder` comment at the bottom

### FILE: compose-templates/.env.example
Content MUST be exactly:

```bash
# .env.example — DevTools environment variable template
# --------------------------------------------------------
# This file is installed to .devtools/.env by the dev-tools skill.
# {{TOKEN}} values are substituted with your choices during installation.
# KEEP .devtools/.env OUT OF GIT — it contains your real credentials.
# The skill adds .devtools/.env to .gitignore on first run.

# ── Project ────────────────────────────────────────────────────────────────────
# COMPOSE_PROJECT_NAME scopes all Docker volume names, network names, and
# container names to your project. Example: myapp → myapp_redis_data volume.
COMPOSE_PROJECT_NAME={{PROJECT_NAME}}

# COMPOSE_PROFILES activates Docker Compose profile groups.
# 'services' activates all main service containers (redis, postgres, etc).
# Add 'ui' or 'monitoring' to also start UI companions or metrics exporters.
# Example: COMPOSE_PROFILES=services,ui
COMPOSE_PROFILES=services


# ── Redis ───────────────────────────────────────────────────────────────────────
# Default port: 6379 | Default version: 7 | Default UI port: 8001
REDIS_PORT={{PORT}}
REDIS_VERSION={{VERSION}}
REDIS_PASSWORD={{PASSWORD}}
REDIS_UI_PORT={{UI_PORT}}


# ── RabbitMQ ────────────────────────────────────────────────────────────────────
# Default AMQP port: 5672 | Management UI port: 15672 | Default version: 3
RABBITMQ_PORT={{PORT}}
RABBITMQ_UI_PORT={{UI_PORT}}
RABBITMQ_VERSION={{VERSION}}
RABBITMQ_USERNAME={{USERNAME}}
RABBITMQ_PASSWORD={{PASSWORD}}


# ── PostgreSQL ──────────────────────────────────────────────────────────────────
# Default port: 5432 | Default version: 16 | Default UI port: 8085
POSTGRES_PORT={{PORT}}
POSTGRES_VERSION={{VERSION}}
POSTGRES_USER={{USERNAME}}
POSTGRES_PASSWORD={{PASSWORD}}
POSTGRES_DB={{DB_NAME}}
POSTGRES_UI_PORT={{UI_PORT}}
# pgAdmin login credentials (for the pgAdmin web UI)
PGADMIN_DEFAULT_EMAIL=admin@local.dev
PGADMIN_DEFAULT_PASSWORD={{PASSWORD}}


# ── MySQL / MariaDB ─────────────────────────────────────────────────────────────
# Default port: 3306 | Default version: 11 (MariaDB) | Default UI port: 8080
# Version 11 = MariaDB 11 (ARM-native). To use MySQL 8, update mysql.compose.yml image line.
MYSQL_PORT={{PORT}}
MYSQL_VERSION={{VERSION}}
MYSQL_ROOT_PASSWORD={{PASSWORD}}
MYSQL_USER={{USERNAME}}
MYSQL_PASSWORD={{PASSWORD}}
MYSQL_DATABASE={{DB_NAME}}
MYSQL_UI_PORT={{UI_PORT}}


# ── MongoDB ─────────────────────────────────────────────────────────────────────
# Default port: 27017 | Default version: 7 | Default UI port: 8081
MONGODB_PORT={{PORT}}
MONGODB_VERSION={{VERSION}}
MONGO_INITDB_ROOT_USERNAME={{USERNAME}}
MONGO_INITDB_ROOT_PASSWORD={{PASSWORD}}
MONGODB_UI_PORT={{UI_PORT}}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env.example → .devtools/.env | SKILL copies and substitutes — no values written until skill runs |
| .devtools/.env → container env | env_file directive loads variables into container runtime |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Information Disclosure | .env.example committed to git | accept | .env.example contains ONLY {{TOKEN}} placeholders and static defaults (admin@local.dev). No real credentials. Real secrets are in .devtools/.env which the skill gitignores. |
| T-01-03-02 | Information Disclosure | PGADMIN_DEFAULT_EMAIL hardcoded default | accept | Default `admin@local.dev` is a non-sensitive placeholder email. Real credential is PGADMIN_DEFAULT_PASSWORD which uses {{PASSWORD}} token. |
| T-01-03-03 | Tampering | COMPOSE_PROFILES=services in .env.example | accept | Static value — activates the services profile by default. User can override by editing .devtools/.env. No privilege escalation risk. |
  | T-01-03-04 | Information Disclosure | Credential variable names exposed in .env.example | accept | Variable NAMES (not values) are visible in .env.example. This is documentation-level exposure; does not reveal actual secrets. |
</threat_model>

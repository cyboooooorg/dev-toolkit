<objective>
Close 4 integration gaps (3 critical breaks + 1 functional gap) identified in the v1.0
milestone audit. Every Taskfile-based service management command is currently broken because
the 6 per-service Taskfile templates still reference pre-Phase-4 flat compose paths. Alias
installs produce unreachable tasks because no include entry is ever appended to
`.devtools/Taskfile.yml`. Alias Taskfile content references a wrong compose path due to an
incomplete substitution rule. And `--profile ui` on a Redis install silently fails because
`REDIS_UI_PORT` is never asked or written to `.env`.

Purpose: Restore all 4 E2E flows to working state; satisfy TMPL-06, MERGE-03, CONF-01.
Output: 6 corrected Taskfile templates + 1 corrected SKILL.md (Steps 6, 10a, 11b, 12).

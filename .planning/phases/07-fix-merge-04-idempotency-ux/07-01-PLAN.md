<objective>
Fix MERGE-04: when the user re-runs the skill with `$ARGUMENTS` equal to an already-installed
alias slug (e.g. `redis-cache`), the skill must emit "redis-cache is already installed — nothing
to do." and exit cleanly — instead of entering the rename-before-add (Step 1a) loop and requiring
the user to type `cancel`.

Purpose: The alias slug path in Step 1 has always been an idempotent intent signal — if the
compose file exists AND the service name is not one of the 6 base names, it IS the intended
service and it IS already installed. The fix adds this discrimination at the `test -f` branch
point in Step 1.

Output: One text edit to `skills/add-service/SKILL.md` — the single line `- If the file **exists** → go to **Step 1a**.` becomes a two-branch conditional that exits early for alias slugs.
</objective>

## Integration Findings

### 🔴 CRITICAL — Functional Breaks (blocks real usage)

#### Break 1 — Per-service Taskfile templates: flat compose path (affects all 4 E2E flows)

**Root cause:** Phase 4 updated `taskfile-templates/root/Taskfile.yml` include paths but did **not** update the 6 per-service Taskfile templates. Phase 4's `04-02-PLAN.md` / `04-02-SUMMARY.md` only lists `skills/add-service/SKILL.md` and `taskfile-templates/root/Taskfile.yml` as modified files.

**Current (broken):**
```yaml
# redis.Taskfile.yml (and all 5 others)
- docker compose -f .devtools/redis.compose.yml up -d --remove-orphans
```

**Correct:**
```yaml
- docker compose -f .devtools/redis/redis.compose.yml up -d --remove-orphans
```

**Impact:** Every `task redis:up`, `task redis:down`, `task redis:logs`, `task redis:restart` fails at runtime. Same for all 5 other services. This is a complete failure of all Taskfile-based service management.  
**Priority:** P0  
**Affects:** TMPL-06, all 4 E2E flows

#### Break 3 — Alias installs: no Taskfile.yml include entry

**Root cause:** SKILL.md Step 11b says "No append is needed for the Taskfile" — correct for the 6 base service names whose `optional: true` entries already exist. For alias installs (redis-cache, redis-session, etc.), no `redis-cache:` include is ever added.

**Impact:** All alias install Taskfile tasks are unreachable. `task redis-cache:up` does not exist.  
**Priority:** P0  
**Fix:** Add instruction in SKILL.md Step 11b: when `MODE=alias`, append `${SERVICE_SLUG}: { taskfile: ${SERVICE_SLUG}/${SERVICE_SLUG}.Taskfile.yml, optional: true }` to `.devtools/Taskfile.yml`.  
**Affects:** MERGE-03

### 🟠 HIGH — Functional Breaks (alias installs)

#### Break 2 — Step 12: incomplete path substitution for alias Taskfile commands

**Root cause:** Step 12 substitution `${SERVICE}.compose.yml → ${SERVICE_SLUG}.compose.yml` renames only the file, not the directory prefix. Result: `.devtools/redis-cache.compose.yml` instead of `.devtools/redis-cache/redis-cache.compose.yml`.

**Impact:** Alias Taskfile task commands reference non-existent file path.  
**Priority:** P1  
**Fix:** Update Step 12 substitution: `${SERVICE}/${SERVICE}.compose.yml` → `${SERVICE_SLUG}/${SERVICE_SLUG}.compose.yml` (replace full relative path, not just filename).  
**Affects:** TMPL-06, MERGE-03

### 🟡 MEDIUM — Partial / Incomplete

#### Gap 1 — REDIS_UI_PORT not asked and not written to .env

`compose-templates/redis/redis.compose.yml` uses `${REDIS_UI_PORT}:5540` for the redisinsight service. `metadata.json` defines `ui_port` (env_var: REDIS_UI_PORT, default: 8001). Neither Step 5 nor Step 6 asks for this port, and Step 10a does not include `REDIS_UI_PORT` in the redis env-var list.

**Impact:** `--profile ui` on a redis install will fail with unresolved `${REDIS_UI_PORT}`.  
**Priority:** P2  
**Affects:** CONF-01

#### Gap 2 — MERGE-04 idempotency UX mismatch

`success_criteria` line 780 states: "Re-running with an identical alias produces 'already installed — nothing to do'." Actual behavior: the conflict loop prompts "Enter a different alias (or 'cancel')" — requires the user to type `cancel`.

**Impact:** UX mismatch; user must type a command to exit the already-installed state.  
**Priority:** P3  
**Affects:** MERGE-04

# Milestone v1.1: Port Management

**Status:** ✅ SHIPPED 2026-04-23
**Phases:** 8–9
**Total Plans:** 2

## Overview

Give users control over host port mapping and prevent port collisions when installing services via the add-service skill. Two phases added opt-in host port binding (Phase 8) and collision detection with a re-prompt escape menu (Phase 9).

## Phases

### Phase 8: Host Port Opt-In

**Goal**: Users decide whether their service needs a host port, and the compose output reflects that choice exactly
**Depends on**: Phase 7 (v1.0 foundation)
**Requirements**: PORT-01, PORT-02
**Plans**: 1 plan

Plans:

- [x] 08-01: Host-port opt-in gate: Q&A question insertion, compose port removal, env var conditional skip, summary display

**Details:**

**Success Criteria** (verified):

1. Skill asks "Do you want to expose this service on a host port?" (default: no) before any port-number question ✅
2. When user says no, the written compose file contains no `ports:` block for that service ✅
3. When user says yes, the skill proceeds to the port-number question ✅
4. A service installed with the no-port path is reachable only within the Docker network — no host port appears anywhere in its compose file ✅

**Implementation:** Inserted `ANSWERS[host_port]` gate in Step 5 Universal Q&A as first question. Port removal uses in-memory modification pattern. Port env var omitted in Step 10a when opted out. Step 8 summary shows `internal only (no host port)`. All 15 decisions (D-01–D-15) implemented.

---

### Phase 9: Port Collision Guard

**Goal**: When a user requests host port binding, their chosen port is validated against all installed services before any file is written, and the skill re-prompts until a safe port is confirmed or the user backs out
**Depends on**: Phase 8
**Requirements**: PORT-03, PORT-04, PORT-05
**Plans**: 1 plan

Plans:

- [x] 09-01: Port scan sub-step + conflict-detection Q2 with 3-option escape menu

**Details:**

**Success Criteria** (verified):

1. Before asking for a port number, skill scans every `.devtools/*/.*compose.yml` and `.devtools/*/.env` to collect all already-bound host ports ✅
2. If the entered port is already bound, skill names the conflicting service and prompts for a different port ✅
3. Re-prompt loop continues until the user enters a non-conflicting port ✅
4. User can opt out of host binding at any point in the re-prompt loop ✅
5. When a non-conflicting port is confirmed, the written compose file contains the correct `ports:` entry ✅

**Implementation:** Port Scan sub-step runs once before Q2 (avoids repeated I/O). `BOUND_PORTS` dict maps port→slug. Conflict message: `Port N is already used by <service> — try M?`. 3-option escape menu: Try different port / Use internal only / Cancel. Option 2 feeds back into Phase 8 opt-out path.

## Milestone Summary

**Key Decisions:**

- Decision: Host-port binding made opt-in (default: internal only) — avoids unexpected host exposure by default
- Decision: Port scan runs once before Q2 prompt, not per retry — minimises file I/O (D-12)
- Decision: Conflict message names the service (`port N is already used by <service>`) — user knows what conflicts
- Decision: 3-option escape menu (Try different port / Use internal only / Cancel) — gives user three clean exits from a conflict
- Decision: Option 2 sets `ANSWERS[host_port]=false`, identical to Phase 8 opt-out — no divergent downstream logic

**Issues Resolved:**

- Automatic port conflict detection added (was in v1.0 Out of Scope as too complex; achieved via compose/env file scanning with zero new dependencies)

**Issues Deferred:**

- Runtime port availability check (`lsof`/`netstat`) — violates zero-dependency constraint; remains out of scope
- Updating ports for already-installed services — separate "update service" flow, not in v1.1

**Technical Debt Incurred:**

- Cancel message "Nothing written." is slightly imprecise on first-run installs — `.devtools/` dir and `.gitignore` written by Step 2 before Step 5 runs. Pre-existing behaviour shared with Phase 8 cancel path; not introduced by v1.1. Non-blocking.

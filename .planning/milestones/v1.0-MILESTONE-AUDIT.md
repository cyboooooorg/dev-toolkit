## Tech Debt

### 🔴 High Priority

**BF-01 — Monitoring Grafana password not applied at runtime**

- **File:** `compose-templates/monitoring/monitoring.compose.yml`
- **Issue:** Grafana service uses `environment: GF_SECURITY_ADMIN_PASSWORD: ${GRAFANA_ADMIN_PASSWORD:-admin}` without `env_file: .env`. SKILL.md Step 10c writes `GRAFANA_ADMIN_PASSWORD` to `.devtools/monitoring/.env`, but Docker Compose YAML interpolation reads from the project root `.env`, not from the service subfolder `.env`. Custom password is silently ignored; fallback `:-admin` always applies.
- **Fix:** Add `env_file: .env` to the `grafana:` service block in `compose-templates/monitoring/monitoring.compose.yml`, consistent with all other service templates.

### 🟡 Medium Priority

**DOC-01 — README "What Gets Written" shows stale flat layout**

- **File:** `README.md`
- **Issue:** "What Gets Written" section shows `.devtools/<service>.compose.yml` (flat, pre-Phase-4). Actual output since Phase 4: `.devtools/<service>/<service>.compose.yml` (subfolder).
- **Fix:** Update the tree block in README.md to reflect the subfolder layout with per-service `.env`.

**WR-01 — Taskfile up tasks may start no containers**

- **Files:** All `taskfile-templates/<svc>/<svc>.Taskfile.yml`
- **Issue:** `task <svc>:up` calls `docker compose` with `--profile services` via COMPOSE_PROFILES env var, but `COMPOSE_PROFILES=services` is in `.devtools/.env`, not the project root `.env`. Docker Compose reads `.env` from CWD. If COMPOSE_PROFILES not exported in shell, zero containers start.
- **Fix:** Either add `dotenv: [.devtools/.env]` to root Taskfile, or pass `--profile services` explicitly (like monitoring already does with `--profile monitoring`).

**WR-02 — up-ui activates UI profile without main service**

- **Files:** All `taskfile-templates/<svc>/<svc>.Taskfile.yml` (up-ui task)
- **Issue:** `up-ui` passes `--profile ui` only. Main service doesn't start unless `--profile services` also activated.
- **Fix:** Change `up-ui` to pass both `--profile services --profile ui`.

### 🟢 Low Priority

**DOC-02 — Compose template headers stale**

- **Files:** All `compose-templates/<svc>/<svc>.compose.yml` (line 2)
- **Issue:** Comment says `# Installed to: .devtools/<service>.compose.yml` (pre-Phase-4 flat path). Correct: `.devtools/<service>/<service>.compose.yml`.
- **Fix:** Update header comment in all 6 compose templates.

**DOC-03 — redis/metadata.json container_port mismatch**

- **File:** `compose-templates/redis/metadata.json` line 13
- **Issue:** `"container_port": 8001` but RedisInsight container uses port 5540. Informational field only — not read by SKILL.md.
- **Fix:** Update `container_port: 5540`.

**WR-03 — RabbitMQ task description inconsistency**

- **File:** `taskfile-templates/rabbitmq/rabbitmq.Taskfile.yml`
- **Issue:** `up` task description references `.env` (project root); all others say `.devtools/.env`.
- **Fix:** Update description to `.devtools/.env` for consistency.
